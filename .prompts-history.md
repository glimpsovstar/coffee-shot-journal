## Phase 1 — V1 espresso journal (empty repo → working app)

### Prompt A — Product brief + README narrative

```
I would like to Create a small React + TypeScript demo app called coffee-shot-journal.

The app should be a personal espresso shot journal. I want to track coffee beans and espresso shots so I can learn what affects consistency and taste over time.

Keep the first version simple:

* a small bean catalogue
* a list of espresso shots
* an add-shot form
* seed data only, no backend
* clean components and readable code

For each bean, track basic reference information:

* name
* roaster
* origin or blend
* roast date
* tasting notes

For each shot, track:

* bean used
* brewed date/time
* grinder and grind setting
* dose in
* yield out
* extraction time
* tasting notes
* rating out of 5

Use local state or simple seed data only. Do not add authentication, database, image upload, weather API, charts, or advanced AI features yet.

Please also create a simple README that explains:

* why I built this
* what the app does today
* what I want to add later
* how this repo can be used to demonstrate an AI-assisted development workflow
```

**Outcome:** Plan mode produced [Coffee Shot Journal v1 plan]; workspace was empty.

## How to reproduce from scratch

1. Create an empty folder and open it in Cursor.
2. Run prompts **A → B** (v1). Verify: `npm run dev`, `npm run build`.
3. Optional: **C** to publish to GitHub.
4. Run **E** (tests). Verify: `npm run test:run`.
5. Run **G** (answer the IndexedDB question if asked) → **H** (photos). Verify: attach photos, refresh browser, photos persist.
6. Compare your git history to the timeline table above.
